ISO 639

Providing localized names for ISO 639 languages

About

The Standard

The ISO 639 standard is commonly used to store language information with unique codes. There are several parts to the standard; the codes for each part are not compatible.

The Service

This service allows you to query an ISO 639 code and receive both the English and localized names. Currently, only part 1 (ISO 639-1) is supported.

Future features may include:

  • Localized names for specific languages (i.e. cross-translated language names), contingent on quality data.
  • Support for other ISO 639 parts, if demand warrants it.

Usage

REST API

Query a language code using a GET request:

/part1?code=pa

And receive a JSON document like this:

{
    "code":"pa",
    "en_name":"Panjabi",
    "alt_en_names":["Punjabi"],
    "local_name":"ਪੰਜਾਬੀ",
    "alt_local_names":["پنجابی‎"]
}

Fields:

code
The ISO 639 code requested
en_name
The English name of the language.
alt_en_names
Alternative English spellings or names (if present). null if does not exist.
local_name
The name of the language as written in the local script.
alt_local_names
Alternative spellings or names as written in local script(s). null if does not exist.
